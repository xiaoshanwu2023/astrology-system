# 星盘查询网站项目修复与完善计划

## 项目概述
- **项目位置**: `~/Desktop/astrology-system/`
- **当前状态**: 基础功能已完成，但存在关键问题需要修复
- **技术栈**: Flask + Python + Pandas + HTML/CSS/JS

---

## 🔴 发现的主要问题

### 1. **astrology.xlsx 数据文件格式问题** (最严重)
- **问题**: 列名包含多余空格（如 `' 星座 '` 而不是 `'星座'`）
- **问题**: 数据量太少（仅8行），无法覆盖所有星座和相位组合
- **问题**: 数据格式不统一，查询逻辑复杂且容易失败
- **影响**: 导致事件查询功能无法正常工作

### 2. **前端页面设计问题**
- **问题**: index.html 文件过大（约14万字符），包含大量冗余代码
- **问题**: 表单设计复杂，用户体验不佳
- **问题**: 缺少响应式设计
- **问题**: 结果页面展示不够直观

### 3. **星盘计算精度问题**
- **问题**: 当前使用简化算法，行星位置计算不够精确
- **问题**: 没有使用专业的星历库（如 flatlib 或 swisseph）
- **问题**: 宫位计算过于简化

### 4. **代码结构问题**
- **问题**: app.py 过于臃肿，缺少模块化
- **问题**: 缺少错误处理机制
- **问题**: 没有日志记录

## 📋 修复计划步骤

### 阶段一：数据文件重构 (优先级：🔴 最高)
**目标**: 重新设计 astrology.xlsx 文件格式，确保数据完整且可查询

1. **分析现有数据**
   - 读取并理解当前8行数据的含义
   - 识别数据模式：星座 + 行星落入 + 相位 → 事件

2. **设计新的数据结构**
   ```
   方案A：规范化表格（推荐）
   - Sheet1: 星座事件 (星座, 事件类型, 事件描述)
   - Sheet2: 行星落入事件 (行星, 星座, 宫位, 事件描述)
   - Sheet3: 相位事件 (行星A, 相位类型, 行星B, 事件描述)
   
   方案B：综合表格
   - 统一表格包含所有查询维度
   ```

3. **生成完整数据**
   - 为12个星座生成基础事件
   - 为10个主要行星 × 12星座 × 12宫位生成落入事件
   - 为主要相位组合生成事件
   - 预计数据量：500-1000行

4. **验证数据可查询性**
   - 确保列名无空格
   - 确保数据格式统一
   - 测试查询逻辑

### 阶段二：后端代码重构 (优先级：🟠 高)
**目标**: 优化代码结构，提高可维护性

1. **模块化重构**
   ```
   app/
   ├── __init__.py
   ├── routes.py          # Flask路由
   ├── calculator.py      # 星盘计算逻辑
   ├── event_lookup.py    # 事件查询逻辑
   └── utils.py          # 工具函数
   ```

2. **改进星盘计算**
   - 考虑集成 flatlib 或 swisseph 库
   - 提高行星位置计算精度
   - 完善宫位计算

3. **增强事件查询功能**
   - 基于重构后的Excel文件重写查询逻辑
   - 添加模糊匹配和相似度匹配
   - 添加默认/通用事件作为备选

4. **添加错误处理和日志**
   - 添加 try-except 块
   - 添加日志记录
   - 添加输入验证

### 阶段三：前端页面重构 (优先级：🟡 中)
**目标**: 提升用户体验和界面美观度

1. **简化首页表单**
   - 减少不必要的字段
   - 优化布局
   - 添加表单验证

2. **美化结果页面**
   - 使用卡片式布局
   - 添加星盘可视化（圆形星盘图）
   - 优化事件展示

3. **添加响应式设计**
   - 适配移动端
   - 适配不同屏幕尺寸

4. **添加加载动画**
   - 计算时显示加载状态
   - 提升用户体验

### 阶段四：功能增强 (优先级：🟢 低)
**目标**: 添加额外功能

1. **星盘可视化**
   - 生成圆形星盘图
   - 显示行星位置和相位线

2. **历史记录**
   - 保存查询历史
   - 支持重新查看

3. **导出功能**
   - 支持导出PDF报告
   - 支持分享结果
